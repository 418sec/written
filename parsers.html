Written

Markdown Syntax Configure Parsers

With Written, you can cherry-pick which parser you want to use. Each instances below were initialized with different parsers.

Markdown Rendered
### Headers and unordered list are accepted - But, the ordered list is not; 1. See, for yourself. 
// Only use headers and ul as block parsers
// Use all inlines parsers
var parsers = new Written.Parsers({blocks: ['header', 'ulist']})
new Written(document.querySelector('.editor'), {parsers: parsers})
Markdown Rendered
### You can't italicize words _Won't work_ but *you can make them bolds* 
// Use all block parsers
// Only use links and bold words as inline parsers
var parsers = new Written.Parsers({inlines: ['link', 'bold']})
new Written(document.querySelector('.editor'), {parsers: parsers})
Markdown Rendered
# No header! *However*, you are free to use _any_ inline parsers you wish. Selfish [promotion](https://github.com/pothibo/written)! Remember, ~~~javascript new Written(document.querySelector('#Editor'))~~~ enables all parsers by default! 
// Use no block parser
// Use all inline parsers
var parsers = new Written.Parsers({blocks: []})
new Written(document.querySelector('.editor'), {parsers: parsers})
Markdown Rendered
### You can also select specific block & inlines! If you really need fine-grained control over what's available. 
// Only use header as block parsers
// Only use links, bold and italic as inline parsers
var parsers = new Written.Parsers({
  blocks: ['header'],
  inlines: ['link', 'bold', 'italic']
})
new Written(document.querySelector('.editor'), {parsers: parsers})
